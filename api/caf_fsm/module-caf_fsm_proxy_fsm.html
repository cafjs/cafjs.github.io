send(self, eventName) → {Promise.<string>}

Source:
  • proxy_fsm.js, line 40

Changes the current state.

After that it triggers the transition actions, and the state listeners functions, all in the same request when we await the send().

In this way, if there is an error, or a failure before checkpointing, both the state change and the actions are ignored.

When the failure is after checkpointing, we always retry the external side-effects of actions and listeners, by using safe transactional plugins.
